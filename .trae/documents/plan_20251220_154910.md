# 解决PaddleClas相对导入错误的计划

## 问题分析
从错误信息来看，问题出在`train_photo_quality.py`的导入语句：
```
from paddleclas import models
```

当直接运行脚本时，Python会将当前目录添加到`sys.path`中，找到当前目录下的`paddleclas.py`文件。但`paddleclas.py`中使用了相对导入（如`from .ppcls.arch import backbone`），而直接运行的脚本没有父包，所以相对导入失败。

## 解决方案
修改训练和推理脚本的导入方式，直接从`ppcls`包中导入所需模块，避免通过`paddleclas.py`进行间接导入。

## 实现步骤

### 1. 修改train_photo_quality.py的导入部分
- 删除`from paddleclas import models`和`from paddleclas.arch import backbone`等导入语句
- 直接从`ppcls`包导入所需模块
- 调整模型创建和使用方式

### 2. 修改infer_photo_quality.py的导入部分
- 同样删除通过`paddleclas`的导入
- 直接从`ppcls`包导入所需模块

### 3. 测试修改后的脚本
- 运行语法检查确保脚本能正常编译
- 验证模型能正确创建和运行

## 预期结果
修改后的脚本能够直接运行，不再出现相对导入错误，并且能正常使用PaddleClas提供的模型和功能。
